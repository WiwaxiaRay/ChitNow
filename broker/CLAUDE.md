# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

The broker is a FastAPI service that intermediates approval requests between AI coding agents (Claude Code / Codex) and an Apple Watch. Core routes and SQLite/SSE logic live in `main.py`; authenticated generic push delivery lives in `relay_client.py`.

## Run & manage

```bash
# First-time setup
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt

# Run directly with generated config, TLS, and restricted permissions
bash run.sh

# Via launchd (auto-starts on login)
launchctl load   ~/Library/LaunchAgents/com.wangyang.thenow-broker.plist
launchctl unload ~/Library/LaunchAgents/com.wangyang.thenow-broker.plist

# Logs
tail -f broker.log

# Quick smoke test
curl -sk https://localhost:8000/health
KEY=$(python3 -c "import json; print(json.load(open('config.json'))['api_key'])")
curl -sk -H "X-API-Key: $KEY" https://localhost:8000/broker-ip
```

## Key design decisions

**SSE for hook blocking**: `POST /approval-requests` creates a DB record and an in-memory `asyncio.Event`. The hook then calls `GET /wait/{id}` which SSE-streams until the event fires or 180 s elapses. On broker restart all in-memory events are lost, so `init_db()` immediately expires any `pending` rows whose `expires_at` has passed.

**Race-condition-free decisions**: `POST /decision/{id}` uses a single atomic `UPDATE … WHERE status='pending' AND expires_at > now` and checks `cursor.rowcount`. If 0 rows updated it distinguishes 404 / 409 / 410 with a follow-up `SELECT`. This prevents two concurrent approve+deny calls both succeeding.

**Relay push**: `relay_client.py` sends only `{"event":"approval_pending"}` to the configured Worker. The Mac never stores the APNs provider private key.

**Usage data**: `GET /usage` reads JSON files written by the [codexbar](https://github.com/steipete/codexbar) menubar app. Four source files, each with a distinct schema — see the inline comments in `main.py`. All reads are wrapped in bare `except Exception` so a missing/malformed file never breaks the response.

## API surface

Normal API routes require `X-API-Key: <api_key>`. `/health` is public; pairing
routes use the localhost setup token and one-time pairing token instead.

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/approval-requests` | Create request, send alert push |
| GET  | `/wait/{id}` | SSE — blocks until decision or timeout |
| POST | `/decision/{id}` | Record approve/deny, fire SSE event |
| GET  | `/pending-requests` | List non-expired pending requests |
| GET  | `/usage` | Claude + GPT token/cost/quota summary |
| POST | `/cancel/{id}` | Cancel a pending request (used by hook on timeout) |
| GET  | `/broker-ip` | Returns `{"url": "https://<mac-ip>:8000"}` |
| GET  | `/approval-routing` | Return authoritative Watch/native approval route |
| PUT  | `/approval-routing` | Atomically persist Watch/native approval route |
| GET  | `/health` | Liveness probe |
| GET  | `/audit` | Last 100 audit log entries |
| POST | `/relay-credentials` | Save relay installation credentials |
| DELETE | `/relay-credentials` | Delete relay installation credentials |
| GET | `/pair?setup_token=...` | Localhost-only pairing page |
| POST | `/pair/{sid}/confirm` | Confirm one-time pairing token |
| GET | `/pair/{sid}/status` | Browser pairing status |

## Gotchas

- The API key is randomly generated by `generate_config.py` and stored in `config.json` (gitignored). Never hardcode or expose it.
- Plain `/pair` requests are intentionally rejected; use the setup-token URL printed by `install.sh`.
- The Broker does not store an APNs provider key. `relay_client.py` sends a generic event to the Worker using `relay_credentials.json`.
- Native approval fallback is allowed only when authenticated `/approval-routing` explicitly returns `watch_approvals_enabled=false`; failures must deny.
- The Mac's LAN IP changes on network switches. If both iPhone and Watch hold a stale URL, the user must re-pair.
